入门教程：从零到第一次联调

面向新加入的开发者与 Agent：按顺序完成依赖、后端联调、架构速览与模块参考，再读编码约定。

1. 克隆与依赖

pnpm install
cp .env.example .env.development.local

在 .env.development.local 设置 VITE_SERVER_BASEURL 指向 mfm-api（如 http://127.0.0.1:8000）。

2. 启动开发服务

# 终端 1：mfm-api
cd ../mfm-api && uv run -m mfm_api

# 终端 2：Admin SPA
pnpm dev

默认 http://localhost:8848。命令与硬约束见仓库根 AGENTS.md §0。

3. 理解架构（Explanation）

  1. C4 Workspace · L1 — 五仓关系
  2. C4 Container · L2 — Admin SPA 边界
  3. C4 Component · L3 — views / api 目录职责
  4. Admin API 边界 — 仅 /admin/*，禁止 C 端 JWT

4. 模块参考（Reference）

按登录 → 菜单 → 业务页的数据流阅读：

  • 鉴权与会话 — login / refresh / setToken
  • 动态菜单与路由 — getAsyncRoutes
  • 集合文档 CRUD — Mongo 白名单表格
  • HTTP 客户端 — Axios 拦截器
  • Pinia 全局状态 — user / permission
  • Layout 壳层 — 侧栏 · 标签页

5. 编码约定（How-to）

阅读 编码约定与护栏 ，重点：路径别名 @/、Admin API 分层、相对 pure-admin 的 MFM 改动清单。

6. 质量闸门

pnpm typecheck
pnpm lint
pnpm test
pnpm verify:harness

详见 测试与验收 · 构建与环境。

7. 第一次改动

建议路径：在 views/business/ 改一个已有页 → 跑 typecheck + lint → 对照 OpenAPI /api/docs（admin tag）自检字段。
