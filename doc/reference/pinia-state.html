Pinia 全局状态

pure-admin 默认 store 模块 + MFM 登录/API 适配。

模块一览

Store 文件 职责
pure-user store/modules/user.ts 用户信息、roles、permissions、登录/登出/refresh
pure-permission store/modules/permission.ts wholeMenus、flatteningRoutes、keep-alive 缓存页
pure-multiTags store/modules/multiTags.ts 多标签页打开的路由栈
pure-settings store/modules/settings.ts 主题、布局模式、界面显示开关
pure-app store/modules/app.ts 侧栏折叠、设备类型等

Hook 用法

组件外或 setup 外使用 useXxxStoreHook()（见各模块底部 export），内部已注入 pinia 实例。 

import { useUserStoreHook } from "@/store/modules/user";
import { usePermissionStoreHook } from "@/store/modules/permission";

与鉴权联动

  • 登录成功 → setToken 写 Cookie/localStorage + user store SET_*
  • refresh → handRefreshToken 更新 token
  • 登出 → 清 user 字段 + permission 缓存 + 重置路由

持久化

用户展示字段通过 storageLocal（user-info）恢复；token 在 Cookie。设置项可走 responsive-storage（见 settings / platform-config）。

业务页建议

  • 集合 CRUD 页状态用组件内 ref / reactive，不必进全局 store
  • 仅跨 layout 需要的会话信息放 user / permission
