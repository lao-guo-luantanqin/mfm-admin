鉴权与会话

pure-admin setToken 方案 + mfm-api Admin 专用登录契约。

登录方式

方式 入口 API 说明
邮箱密码 登录页表单 POST /admin/auth/login username 字段值为邮箱
开发虚拟登录 登录页按钮（DEV） POST /admin/auth/dev-login 需 VITE_DEV_MOCK_LOGIN=true；后端 dev 环境 superuser

前端文件

  • src/api/user.ts — getLogin、getDevLogin、refreshTokenApi
  • src/store/modules/user.ts — loginByUsername、loginByDev、handRefreshToken、logOut
  • src/utils/auth.ts — getToken / setToken / removeToken
  • src/views/login/index.vue — UI 与虚拟登录开关

Token 存储

存储 Key 内容
Cookie authorized-token accessToken、expires、refreshToken
localStorage user-info 头像、用户名、roles、permissions 等
Cookie multiple-tabs 多标签会话标记

无感 refresh：见 HTTP 客户端 专题。

角色与权限

登录响应一次返回 roles（页面级）与 permissions（按钮级），写入 Pinia user store。

后端门禁：is_superuser、MFM_API_ADMIN_ALLOWED_EMAILS 等（见 mfm-api services/admin/auth/access.py）。

登出

useUserStore().logOut()：清 token、重置路由与标签页、跳转 /login。当前为前端登出，不调用后端 revoke 接口。
