Layout 壳层

vue-pure-admin 布局系统；MFM 业务逻辑应放在 views/business，而非改 layout 核心。

结构

src/layout/index.vue
├── lay-sidebar/     侧栏菜单（wholeMenus）
├── lay-navbar/      顶栏
├── lay-tag/         多标签页
├── lay-content/     <router-view> + keep-alive
├── lay-search/      菜单搜索
├── lay-setting/     主题与布局设置
└── lay-footer/

platform-config

  • 静态：public/platform-config.json
  • 启动时 getPlatformConfig(app)（src/config/index.ts）合并到全局 $config
  • 控制 Title、布局模式、是否显示 footer 等 pure-admin 行为

通用组件（Re*）

src/components/Re* — pure-admin 封装：表格栏、权限、图标、对话框等。业务页可直接使用 Element Plus + RePureTableBar。

iframe 路由

外链菜单走 layout/frame.vue；当前 MFM 业务均为本地 SFC，未使用 iframe 菜单。

样式

  • 全局：src/style/index.scss、element-plus.scss、tailwind.css
  • 侧栏：sidebar.scss；暗色：dark.scss
  • 业务页优先 scoped SCSS，避免污染 layout

修改原则

  • 品牌/logo：改 SidebarLogo、platform-config
  • 不要为单一业务集合改 layout 数据流
  • 升级 pure-admin 基线时，优先保持 layout 与 upstream diff 最小
