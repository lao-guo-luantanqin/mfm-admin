动态菜单与路由

静态路由 + 后端 GET /admin/menus 驱动的 pure-admin 动态路由。

流程

  1. 登录成功 → 路由守卫调用 initRouter()（src/router/utils.ts）
  2. getAsyncRoutes() → GET /admin/menus
  3. handleAsyncRoutes 将 component 字符串映射为 import.meta.glob('/src/views/**/*.{vue,tsx}')
  4. usePermissionStore().handleWholeMenus(routes) 生成侧栏菜单

关键文件

  • src/api/routes.ts — HTTP 封装
  • src/router/index.ts — 守卫、NProgress、token 检查
  • src/router/utils.ts — initRouter、排序、权限过滤
  • src/router/modules/*.ts — 静态路由（home、error 等）
  • src/store/modules/permission.ts — wholeMenus、keep-alive 缓存列表

后端菜单注册

菜单树由 mfm-api services/admin/menus/registry.py · admin_async_routes() 返回，示例： 

{
  "path": "/business/collections/index",
  "name": "BusinessCollections",
  "component": "business/collections/index",
  "meta": { "title": "集合文档", "roles": ["admin", "editor"] }
}

component 对应 src/views/business/collections/index.vue。

系统管理（RBAC）：system/roles/index、system/operators/index，仅 meta.roles: ["admin"] 可见。

新增菜单页

  1. 在 src/views/ 下新增 SFC，设置 defineOptions({ name: "..." })
  2. 在后端 admin_async_routes() 增加 child 节点
  3. 本地重启 mfm-api，刷新管理端验证侧栏与路由
  4. 若需按钮权限，在 meta 或页面内配合 permissions

静态 vs 动态

类型 来源 示例
静态 router/modules/ 首页、错误页、remaining（login）
动态 /admin/menus 业务数据、集合文档
