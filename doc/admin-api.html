Admin API 边界

管理端仅调用 mfm-api 前缀 /admin/*；字段级契约以 OpenAPI /api/docs（tag: admin）为准。

禁止：直连 C 端 /api/property/* 等做运营写操作；禁止在前端复刻 sync-batch 批量协议。

响应信封

与 pure-admin 习惯一致：顶层 { success, data, msg? }。HTTP 层在 utils/http 对 success === false 弹错并 reject。

登录会话 data 字段（camelCase，由后端 OutBase 序列化）：accessToken、refreshToken、expires、username、nickname、avatar、roles、permissions。

分页查询参数

集合列表使用 pure-admin / Element 习惯别名：

  • currentPage — 页码，从 1 开始
  • pageSize — 每页条数，后端上限 200
  • keyword — 可选全文检索

响应 data.list、data.total、data.pageSize、data.currentPage。

后端白名单

可管理集合由 mfm-api 环境变量控制，常见：

MFM_API_SYNC_BATCH_ALLOWED_COLLECTIONS=mfm_property_project,mfm_banner,...

未在白名单的集合：meta 接口不返回或 CRUD 返回 403/404（以后端为准）。
