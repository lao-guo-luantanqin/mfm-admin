系统架构

济南买房么运营后台基于 vue-pure-admin-thin，通过专用 Admin API 管理 Mongo 业务集合。

设计原则：pure-admin 定交互，mfm-api /admin/* 适配——不复用 C 端 JWT 或 clientDB。

技术栈

层 选型 说明
框架 Vue 3 + TypeScript Composition API、<script setup>
构建 Vite 7 ESM、import.meta.env
UI Element Plus + Tailwind 4 表格/表单以 Element 为主
壳层 pure-admin layout 侧栏、多标签、权限指令
状态 Pinia user / permission / multiTags 等
HTTP Axios（utils/http） 唯一客户端，Bearer + refresh
后端 mfm-api FastAPI 仅 /admin/* 管理契约

请求生命周期

  1. 用户访问 → 路由守卫检查 getToken()
  2. 无 token → /login；有 token → initRouter() 拉动态菜单
  3. 业务页调用 src/api/admin/* → http 注入 Bearer
  4. 401/403 → 清 token，回登录；access 过期 → 队列 refresh 后重试

目录分层（摘要）

目录 职责
src/api/ Admin HTTP 封装（薄层）
src/views/business/ MFM 业务页面（当前：collections）
src/layout/ pure-admin 壳，一般不改业务逻辑
src/router/ 静态路由 + 动态路由装配
src/store/modules/ 会话、菜单、标签页状态

完整目录真值见仓库根 AGENTS.md §1。

与兄弟项目关系

项目 关系
