构建与环境

Vite 多模式环境变量与本地联调步骤。

快速联调

# 1. 后端
cd ../mfm-api
uv run -m mfm_api

# 2. 管理端
cd ../mfm-admin
cp .env.example .env.development.local
pnpm install
pnpm dev

浏览器打开 http://localhost:8848

环境变量

变量 说明
VITE_SERVER_BASEURL mfm-api 根 URL，如 http://127.0.0.1:8000
VITE_DEV_MOCK_LOGIN true 时登录页显示虚拟登录
VITE_PUBLIC_PATH 部署子路径（见 .env.production）
VITE_ROUTER_HISTORY hash 或 h5 模式

本地覆盖：复制 .env.example → .env.development.local（勿提交密钥）。

构建命令

命令 用途
pnpm dev 开发服务器 :8848
pnpm build 生产构建 → dist/
pnpm build:staging staging 模式构建
pnpm preview 预览 dist

后端依赖配置

mfm-api .env 至少配置：

MFM_API_ENVIRONMENT=dev
MFM_API_SYNC_BATCH_ALLOWED_COLLECTIONS=mfm_property_project,mfm_banner,...

开发虚拟登录需 MFM_API_ENVIRONMENT=dev（或非 production）。

Docker

仓库含 Dockerfile；编排通常由 DAS / mfm-datacenter compose 引用。生产 VITE_SERVER_BASEURL 在构建时注入，指向对外 API 域名。
